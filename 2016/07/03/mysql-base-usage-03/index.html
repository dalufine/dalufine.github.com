《mysql 必知必会》读书笔记三（21-30）

mysql
  • note

创建表

每个表中只允许有一个AUTO_INCREMENT列，而且该列必须被索引（如，使他成为主键），该列的值并没有特殊的格式要求，只要是唯一就行。

select last_insert_id()可以返回最后一个AUTO_INCREMENT的值。

引擎

  1. InnoDB是一个可靠的事务处理引擎，它不支持全文本搜索；
  2. MEMORY在功能等同于MyISAM，但由于数据存储在内存，速度很快，特别适用于临时表；
  3. MyISAM是一个性能极高的引擎，它支持全文本搜索，但不支持事务处理。

用同一个引擎的表不能引用具有使用不同引擎的表的外键。

视图

视图仅仅是用来查看存储在别处的数据的一种设施，其本身不包含数据，因此它返回的数据都是从其他表中检索出来的。

存储过程

存储过程简单的来说可以理解为：为以后的使用而保存的一条或者多条MySql语句的集合，有点类似于批处理。


demo

游标

触发器

insert

  1. insert触发器代码内，可以引用一个名为NEW的虚拟表，访问被插入的行；
  2. 在before insert触发器中，NEW中的值也可以被更新（即允许修改被插入的值）；
  3. 对于auto_increment列，NEW在insert执行之前包含0，在insert执行后包含新的自动生成的值。

delete

  1. delete触发器内部，你可以引用一个名为OLD的虚拟表，访问被删除的行；
  2. OLD中的值全部是只读的，不能更新。

update

  1. 在update触发器代码中你可以引用一个名为OLD的虚拟表访问（update语句前）的值，引用一个名为NEW的虚拟表访问新的更新值；
  2. 在before update触发器中，NEW中的值可以被更新；
  3. OLD中的值全部为只读。
